## Мета проєкту

Розробити повноцінний ML-проєкт, який покаже ваші професійні навички та стане конкурентною перевагою для портфоліо. Ви маєте самостійно пройти всі етапи роботи з даними – від дослідницького аналізу до побудови, оцінювання та інтерпретації моделей машинного навчання.

**Цей проєкт стане ідеальним способом показати роботодавцю, що ви не лише маєте знання, але й можете їх ефективно застосовувати** 💪🔥

## Мінімальні вимоги до задачі проєкту

Задача фінального проєкту – підсумувати ваші знання в кінці навчання і створити справді крутий проєкт, який вигідно доповнить ваш професійний профіль і допоможе в отриманні роботи. Виходячи з цього маємо кілька вимог до фінального проєкту:

### 1. Тип задачі

**Задача має бути supervised learning:**

- **Класифікація** (binary або multi-class)
- **Регресія**
- **Рекомендаційні системи** (з чіткою метрикою оцінки)

Задачі кластеризації або інші unsupervised підходи **НЕ підходять** для фінального проєкту, адже нам треба чітке порівняння моделей на метриках точності.

### 2. Тип даних

Дані **обов'язково мають містити неструктуровані форми даних:**

- **Текст** (описи, відгуки, статті, твіти, питання тощо)
- **Зображення**
- **Аудіо** (опціонально)

❌ Дані **НЕ можуть бути просто табличними** лише з категоріальними і числовими ознаками.

✅ Якщо дані **табличні і містять текст** (назви статей/твіти/великі тексти/описи продуктів) – нам це **підходить** 👌

### 3. Розмір датасету

- **Мінімум 2000 екземплярів** (це самий мінімум для табличних та текстових даних)
- **Бажано >10000 записів** для більш репрезентативних результатів
- Для відео-файлів може бути менше екземплярів

### 4. Моделі

В якості рішення мають бути **протестовані мінімум 3 різних моделі**.

**Приклади комбінацій моделей:**

1. LLM за API (наприклад, OpenAI, Claude, Gemini)
2. XGBoost / LightGBM / CatBoost
3. Fine-tuned BERT / RoBERTa / DistilBERT

**Інші варіанти:**

- Logistic Regression (baseline)
- Random Forest
- Neural Networks (PyTorch/TensorFlow)
- Sentence Transformers для текстів
- CNN для зображень
- Будь-які інші моделі, які ви вивчили або хочете спробувати

✅ Можна використовувати **як моделі, які ми вивчили, так і моделі, які не були розглянуті в курсі** 👌

**Увага!** Для бінарної класифікації логічним бейзлайном буде заповнити всі передбачення більш частотним класом, але це НЕ вважається одною з трьох моделей з експериментів. 

### 5. Оформлення

Фінальний проєкт **має бути оформлений на GitHub** з детальним **README.md.** Детальніше про оформлення - нижче.

## Що обов'язково треба зробити у проєкті

### 1. Аналіз вашого профілю

Проаналізуйте 2-3 вакансії ML Engineer / Data Scientist, які вам цікаві, та визначте:

- Які навички найбільш затребувані - саме ці навички варто продемонструвати в проєкті
- Які типи задач часто згадуються - виходячи з цього радимо обрати задачу
- Сформулюйте на основі аналізу, що ви хочете продемонструвати у проєкті

### 2. Дослідницький аналіз даних (EDA)

**Обов'язково:**

- Опис структури даних
- Аналіз розподілу цільової змінної
- Виявлення пропущених значень та викидів
- Візуалізація ключових закономірностей
- **Мінімум 2 інформативних графіки**

### 3. Підготовка даних

Feature Engineering:

- **Для текстових даних: т**окенізація, векторизація (BoW, TF-IDF, embeddings)
- **Для зображень:** data preprocessing, data augmentation
- **Для табличних даних:**
    - Створення нових ознак (якщо застосовно)
    - Обробка пропущених значень
    - Кодування категоріальних змінних

Для всіх типів даних:

- Розділення на train/validation/test

### 4. Оцінювання моделей

Необхідно обрати **1 основну метрику** (на її основі прийматиметься рішення про те, яка модель є найкращою) і можна додати ще 2 допоміжних. В цілому, кількість метрик - не обмежена.

### 5. Baseline модель

Побудуйте просту baseline модель, щоб мати точку відліку:

- Для класифікації: Logistic Regression або найчастіший клас
- Для регресії: Linear Regression або середнє значення
- Оцініть результати baseline

### 6. Побудова та порівняння моделей

**Мінімум 3 різних моделі:**

- Побудуйте кожну модель
- Доберіть гіперпараметри (Grid Search, Random Search, Optuna)
- Оцініть на validation set
- Порівняйте результати

**Створіть таблицю експериментів - приклад:**

| Модель | Параметри | Train Score | Validation Score | Test Score | Час тренування |
| --- | --- | --- | --- | --- | --- |
| Baseline | - | 0.65 | 0.64 | - | 1 сек |
| XGBoost | max_depth=5, n_estimators=100 | 0.89 | 0.82 | - | 5 хв |
| BERT fine-tuned | lr=2e-5, epochs=3 | 0.92 | 0.86 | 0.85 | 2 год |

### 7. Аналіз результатів

- Feature importance (які ознаки найважливіші?)
- Аналіз помилок (де модель помиляється?)
- Візуалізація результатів (confusion matrix, ROC-крива, residual plots)
- Інтерпретація передбачень (чому модель приймає такі рішення?)

**Не обовʼязково робити всі пункти, але хоча б один має бути присутнім.**

### 8. Висновки

- Яка модель працює найкраще і чому?
- Які практичні інсайти отримали?
- Як модель може використовуватися в бізнесі?
- Які обмеження у моделі?
- Що можна покращити далі?

Можна описати в Jupyter Notebook або в README проєкту.

### 9. Деплоймент

Опціонально можна розгорнути проєкт на Streamlit/REST API

### 10. Оформлення проєкту в GitHub репозиторії

**Приклад структури репозиторію:**

```
project-name/
├── README.md              # Детальний опис проєкту
├── requirements.txt       # Залежності
├── data/                  # Дані (або посилання на них)
├── notebooks/             # Jupyter notebooks з EDA та експериментами
│   ├── 01_eda.ipynb
│   ├── 02_baseline.ipynb
│   ├── 03_models.ipynb
├── src/                   # Код для моделей
│   ├── preprocessing.py
│   ├── models.py
│   ├── evaluation.py
├── models/                # Збережені моделі
└── reports/               # Звіти, графіки

```

**README.md має містити:**

1. Назва проєкту та короткий опис
2. Бізнес-задача та мета
3. Дані (посилання на джерело, розмір, короткий опис)
4. Підхід до оцінювання, обрана метрика.
5. Підхід до розв'язку, інструменти
6. Результати (таблиця експериментів)
7. Висновки
8. Як запустити проєкт (Installation & Usage)
9. Вимоги (requirements.txt)

### 11. Відео-презентація

**Запишіть коротке презентаційне відео в Loom (3-5 хв)** англійською або українською мовами:

- Яку задачу вирішували
- Які дані використовували
- Які моделі побудували та порівняли
- Які результати отримали
- Практичне застосування

Це тренування для співбесід!
